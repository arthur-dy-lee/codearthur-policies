## 1 产品愿景与定位

### 1.1 产品一句话描述

一款帮助中国职场人群规划财务自由之路的工具类 App。通过可视化的动态模拟，让用户清晰的看到"我距离财务自由还有多远"，从而缓解职场焦虑，获得对生活的掌控感。

### 1.2 核心价值主张

> **JTBD 核心洞察**
>
> 用户不是在找一个"财务计算器"，而是在雇佣一个"财务自由可视化指南针"来缓解焦虑、提供确定性、获得行动方向。

- **贩卖确定性：** 把模糊的"我要存钱"变成清晰的"再坚持 7 年 3 个月，我就自由了"
- **提供情绪价值：** 每次打开 App 看到进度增长，获得"我在接近目标"的正反馈
- **降低行动门槛：** 不需要金融知识，录入基础数据就能获得专业级规划

### 1.3 目标用户画像

| 维度 | 描述 |
|------|------|
| 年龄 | 25-40 岁职场人群 |
| 收入 | 月薪 8K-50K，有一定储蓄但规划模糊 |
| 痛点 | 职场内卷焦虑，想知道"我到底还要干多久" |
| 行为特征 | 关注 FIRE 社区（豆瓣/即刻/小红书），有基础理财意识 |
| 技术偏好 | 习惯移动端，偏好简洁直观的界面 |

## 2 JTBD 用户故事与场景分析

### 2.1 核心 Job Statement

> **Job Statement**
>
> 当我在内卷环境中感到疑惑和焦虑时，我想要清晰地看到我的财务自由进度和具体路径，以便我能缓解焦虑、做出理性决策，并获得对未来的掌控感。

### 2.2 用户故事列表

以下采用标准用户故事格式：作为 [ 角色 ]，我想要 [ 功能 ]，以便 [ 价值 ]。

#### US-1 快速查看 FIRE 进度

作为 **职场人**，我想要 **打开 App 立刻看到"距离财务自由还有 X 年 X 个月"的动态数字**，以便 **瞬间获得明确的目标感和动力**。

#### US-2 动态模拟退休场景

作为 **有储蓄的年轻人**，我想要 **调整储蓄率、投资收益率、生活开支等参数，实时看到 FIRE 时间线的变化**，以便 **找到最适合我的财务自由路径**。

#### US-3 记录每月财务快照

作为 **FIRE 追赶者**，我想要 **每月快速录入收入、支出、净资产数据**，以便 **追踪进度是否偏离计划，及时调整策略**。

#### US-4 可视化财富增长曲线

作为 **需要动力的用户**，我想要 **看到我的资产随时间复利增长的可视化图表**，以便 **直观感受"复利的力量"，增强坚持的信心**。

#### US-5 多种 FIRE 类型对比

作为 **FIRE 新手**，我想要 **了解 Lean FIRE / Regular FIRE / Fat FIRE 的区别和各自所需资金**，以便 **选择适合我生活方式的 FIRE 目标**。

#### US-6 "如果我..."假设探索

作为 **对未来充满不确定的用户**，我想要 **模拟"如果我涨薪/跳槽/副业收入增加"等场景对 FIRE 时间的影响**，以便 **做出更理性的职业和财务决策**。

### 3.2 核心功能详细设计

#### 模块一：FIRE 进度看板

对应 User Story：**US-1**

这是用户打开 App 后的第一个画面，也是整个产品的"情绪键"。

- **核心数字：** 居中大字显示"距离财务自由还有 X 年 X 个月"
- **进度环：** 圆形进度条显示当前资产 / 目标资产的百分比
- **动态数据卡片：** 当前净资产、FIRE 目标额、月储蓄率、预估退休年份
- **每日一句话：** 根据进度生成励志文案，如"你已经走完了 23% 的路，继续加油！"

#### 模块二：基础财务数据录入

对应 User Story：**US-3**

降低录入门槛是留存的关键。设计原则：能不录就不录，能默认就默认。

- **最简录入：** 只需 4 个数字——月收入、月支出、当前净资产、期望退休年支出
- **智能默认值：** 投资收益率默认 7%（沿用指数基金历史平均），通胀率默认 3%
- **渐进式引导：** 首次使用只要求必填项，后续引导用户补充更详细的数据

#### 模块三：FIRE 动态模拟器

对应 User Story：**US-2, US-6**

这是产品的核心引擎，用户通过调参数探索不同的 FIRE 路径。

- **可调参数：** 月储蓄率、投资收益率、通胀率、目标年支出、当前年龄
- **滑动条交互：** 拖动滑动条实时看到图表和数字变化，体验"每多存 1000 元的影响"
- **场景对比：** 保存多个模拟场景，并排对比（如"保守版 vs 激进版 vs 副业版"）
- **计算核心：** 基于 4% 规则（年支出 × 25 = 目标资产）+ 复利计算引擎

#### 模块四：复利增长可视化

对应 User Story：**US-4**

- **财富增长曲线：** 显示从现在到 FIRE 日的资产增长轨迹，用面积图区分"本金"与"投资收益"
- **里程碑标记：** 在曲线上标注关键节点（资产达 100万、储蓄超过支出、资产达到 FIRE 数字的 50%）
- **"复利拐点"高亮：** 标注投资收益超过本金投入的时间点，这是最具励志意义的瞬间

## 4 信息架构与交互设计

### 4.1 应用导航结构

采用底部导航栏 Tab 设计，4 个核心页面：

| Tab | 名称 | 核心内容 | 对应 Job |
|:---:|------|---------|---------|
| 1 | 看板 | FIRE 进度数字 + 复利曲线 | "我距离自由多远" |

| 4 | 我的 | 个人设置 + FIRE 类型 + 数据管理 | 个性化配置 |

### 4.2 用户首次体验流程 (Onboarding)

设计原则：30 秒内让用户看到"哇！"的感觉。

1. **欢迎页：** "来算算你的财务自由日吧"（情绪钩子）
2. **快速录入：** 只需填 4 个数字（月收入/支出/净资产/目标年支出）
3. **"哇"时刻：** 立即展示"你距离财务自由还有 X 年"的动态数字
4. **引导探索：** "试试调整参数，看看能不能更快？"

## 5 核心算法模型

### 5.1 FIRE 目标额计算（4% 规则）

> **公式**
>
> FIRE 目标资产 = 年度目标支出 ÷ 4% = 年度目标支出 × 25

例如：如果退休后每年想花 12 万元，则需要积累 300 万元资产。

### 5.2 FIRE 时间计算引擎

每月迭代计算，考虑以下变量：

- **月储蓄额：** 月收入 - 月支出
- **投资收益：** 现有资产 × 月化收益率
- **通胀调整：** 目标支出每年按通胀率增长
- **终止条件：** 当累计资产 ≥ FIRE 目标额时，达到 FIRE

### 5.3 参数范围与默认值

| 参数 | 默认值 | 可调范围 | 说明 |
|------|:------:|---------|------|
| 月收入 | 无默认 | 1K - 500K | 用户必填 |
| 月支出 | 无默认 | 0 - 500K | 用户必填 |
| 当前净资产 | 0 | 0 - 5000万 | 含存款+投资 |
| 年化投资收益率 | 7% | 1% - 15% | 沿用指数基金历史均值 |
| 通胀率 | 3% | 0% - 8% | 中国 CPI 参考 |
| 安全提取率 | 4% | 3% - 5% | Trinity Study 经典值 |
| 当前年龄 | 30 | 18 - 60 | 用于计算退休年龄 |

## 6 非功能需求

### 6.1 性能要求

- **首屏加载：** 冷启动 < 2秒，热启动 < 0.5秒
- **模拟计算：** 滑动条拖动时图表实时更新，延迟 < 100ms
- **离线可用：** 核心功能无需网络，所有计算本地完成

### 6.2 数据安全

- **本地存储优先：** 财务数据存储在本地，不强制上传云端
- **可选云同步：** 支持 iCloud/阿里云同步，端到端加密
- **无账号使用：** MVP 版本不强制注册，降低使用门槛

### 6.3 技术架构建议

| 层面 | 推荐方案 | 原因 |
|------|---------|------|
| 跨平台框架 | Flutter / React Native | 独立开发者效率优先，一套代码双端 |
| 本地存储 | SQLite + Hive/MMKV | 轻量、快速、离线可用 |
| 图表库 | fl_chart / Victory Native | 流畅动画 + 交互式图表 |
| 状态管理 | Riverpod / Zustand | 轻量、易维护 |
| 后端（可选） | Supabase / Firebase | 后期云同步、用户系统 |

## 7 商业模式与增长策略

### 7.1 变现策略：免费增值 (Freemium)

| 层级 | 功能 | 价格 |
|------|------|------|
| 免费版 | 进度看板 + 基础模拟器（1个场景）+ 复利曲线 | 免费 |
| Pro 版 | 无限场景对比 + "如果我..."模块 + FIRE 类型分析 + 数据导出 | ¥18/月 或 ¥168/年 |
| Pro+ 版 | 投资组合建议 + 蓝牙合算 + 财务报告生成 | ¥28/月 或 ¥268/年 |

### 7.2 增长策略

- **种子用户：** 在豆瓣 FIRE 小组、即刻 FIRE 话题、小红书理财区发布 Demo 视频
- **裂变机制：** "分享你的 FIRE 进度图"生成精美的社交分享卡片，带二维码下载链接
- **SEO/ASO：** 关键词覆盖"FIRE 财务自由""退休计算器""理财规划"等搜索意图
- **内容营销：** 开设"FIRE 知识库"板块，提供免费的 FIRE 科普内容吸引自然流量
